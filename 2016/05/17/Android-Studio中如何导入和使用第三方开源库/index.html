Android Studio中如何导入和使用第三方开源库

Android Studio中如何导入和使用第三方开源库?

第三方的项目分为以下几种类型:

  • 第一种:第三方远程开源库
  • 第二种:第三方本地开源库(module)
  • 第三种:第三方jar包

导入并使用远程开源库

比如github中很多开源项目,直接给你一个Maven地址,这是最容易的一种,直接一句话搞定
只需要在你的项目(module)中的build.gradle文件的dependence中加入一个远程依赖库地址即可,studio自动去帮你下载远程依赖库.
第一步:在build.gradle文件中添加远程依赖

第二步:同步gradle

第三步:当gradle同步完成不报错的情况下,按Ctrl+F9重新编译项目,因为同步只是去帮你下载,下载完成后你在项目中直接使用可能找不到文件,
重新编译一下文件之后就能在项目中直接关联了.

第三方本地开源库(module)

如果是第三方开源库没有提供远程库只提供了源码module,那么就需要使用这种方法进行导入了,
这种方法可能要注意的是本地开源库中也包含gradle文件,gradle中可能依赖了你的项目中冲突或者没有的远程库,这种情况下可能会报错.
一般报错后检查和修改本地开源库中的gradle文件即可.
第一步:
在你的project下创建文件夹(注意是project不是module),并将本地开源库拷贝到你创建的文件中,比如我创建了一个叫library的文件夹

注意:红线部分文件夹的样子,当完成依赖以后图标才会变成这个样子!
第二步:
给项目添加依赖

  • 1.修改project项目下的settings.gradle 文件
    添加:include ‘:App’,’:library’ library 就是你的新建的文件夹的名字
  • 2.修改你项目中的build.gradle
    添加:
    dependencies{
    compile project(‘:library ‘)

}

OK,现在gradle同步一下就可以了,同步完成最好再编译一下.

导入第三方jar包

导入第三方jar包的方式和导入本地开源库的方式差不多
第一步也是新建文件夹,将jar包复制到文件夹中.剩下的就是在gradle文件中写法可能略有不同.
在module文件夹下修改build.gradle 文件
compile files(‘文件夹名/jar包名.jar’)
